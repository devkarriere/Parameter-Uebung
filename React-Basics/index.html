React Basics

Komponenten, Props, State & Callbacks – deine ersten Schritte mit React

1. Was ist eine Komponente?

Stell dir Lego-Bausteine vor. Jeder Baustein hat eine bestimmte Form und Farbe. Du kannst sie immer wieder verwenden und zusammenstecken. React-Komponenten sind wie digitale Lego-Bausteine für deine Webseite.

Eine Komponente ist eine Funktion, die sichtbare Elemente zurückgibt. Diese spezielle Schreibweise heißt JSX – sie sieht aus wie HTML, ist aber JavaScript. 

// Eine einfache Komponente
function Greeting() {
  return <h1>Hallo, Welt!</h1>;
}

// So wird sie im Browser angezeigt:
const root = ReactDOM.createRoot(document.getElementById("root"));
root.render(<Greeting />);

Wichtig: Der Funktionsname muss mit einem Großbuchstaben anfangen. So weiß React, dass es sich um eine Komponente handelt und nicht um ein normales HTML-Element.

App (Eltern-Komponente) <Greeting />
Greeting (Kind-Komponente) <h1>Hallo, Welt!</h1>
Merke: Eine Komponente ist eine Funktion, die JSX zurückgibt. JSX sieht aus wie HTML, ist aber JavaScript. Der Funktionsname muss mit einem Großbuchstaben anfangen.

2. Props – Daten an Komponenten übergeben

Du kennst schon Parameter bei Funktionen. Props sind das Gleiche – nur für Komponenten! Wenn die Komponente ein Briefkasten ist, sind Props die Briefe, die du hineinwirfst.

Props (kurz für „Properties“, also Eigenschaften) werden beim Verwenden der Komponente übergeben. Die Komponente empfängt sie als ein einziges Objekt. 

function Greeting(props) {
  return <h1>Hallo, {props.name}!</h1>;
}

// Aufruf mit verschiedenen Props:
<Greeting name="Anna" />   // zeigt: Hallo, Anna!
<Greeting name="Max" />    // zeigt: Hallo, Max!

Statt jedes Mal props.name zu schreiben, kannst du die Werte direkt in der Klammer „auspacken“. Das nennt man Destrukturierung – du holst dir genau die Eigenschaften heraus, die du brauchst: 

// Vorher: props als ganzes Objekt
function Greeting(props) {
  return <h1>Hallo, {props.name}!</h1>;
}

// Nachher: direkt destrukturiert – kürzer und übersichtlicher
function Greeting({ name }) {
  return <h1>Hallo, {name}!</h1>;
}

// Funktioniert genauso mit mehreren Props:
function ProfileCard({ name, age, city }) {
  return (
    <div>
      <h2>{name}</h2>
      <p>Alter: {age}</p>
      <p>Stadt: {city}</p>
    </div>
  );
}
Merke: Die geschweiften Klammern { } in function Greeting({ name }) sind Destrukturierung. Du sagst damit: „Nimm aus dem Props-Objekt die Eigenschaft name und speichere sie direkt als Variable.“ Das Ergebnis ist das Gleiche – nur kürzer. In der Praxis wirst du fast immer diese Schreibweise sehen.
name="Anna" Attribut → props.name "Anna" → {props.name} im JSX

Das Attribut name wird zum props.name in der Komponente.

Du kannst auch mehrere Props übergeben:

function ProfileCard(props) {
  return (
    <div>
      <h2>{props.name}</h2>
      <p>Alter: {props.age}</p>
      <p>Stadt: {props.city}</p>
    </div>
  );
}

// Texte in Anführungszeichen, Zahlen in geschweiften Klammern:
<ProfileCard name="Lisa" age={28} city="Berlin" />
Merke: Props werden wie HTML-Attribute übergeben. Texte stehen in Anführungszeichen, Zahlen und Variablen in geschweiften Klammern {}. Props sind read-only – die Komponente darf sie nicht verändern.

3. Bedingte Anzeige – Inhalte dynamisch anpassen

Manchmal soll eine Komponente je nach Situation etwas anderes anzeigen. Das kennst du schon von if/else – in React geht das genauso! Dazu gibt es zwei Wege:

Weg 1: Der Ternär-Operator (bedingung ? wennJa : wennNein) – perfekt für kurze Entscheidungen direkt im JSX: 

function StatusBadge(props) {
  return (
    <span className={props.isOnline ? "badge-green" : "badge-gray"}>
      {props.isOnline ? "Online" : "Offline"}
    </span>
  );
}

<StatusBadge isOnline={true} />    // grüner Badge "Online"
<StatusBadge isOnline={false} />   // grauer Badge "Offline"

Weg 2: if/else mit Variablen – besser für längere Entscheidungen: 

function WeatherIcon(props) {
  let icon;
  let color;

  if (props.temperature > 25) {
    icon = "☀️";
    color = "text-warm";
  } else if (props.temperature > 10) {
    icon = "⛅";
    color = "text-mild";
  } else {
    icon = "❄️";
    color = "text-cold";
  }

  return (
    <div className={color}>
      <span>{icon}</span>
      <span>{props.temperature}°C</span>
    </div>
  );
}

Falsch:

// class ist ein reserviertes Wort in JavaScript!
<div class="badge-green">Online</div>

Richtig:

// In JSX: className statt class
<div className="badge-green">Online</div>
Achtung: class ist ein reserviertes Wort in JavaScript. Deshalb heißt es in JSX immer className!
Merke: Mit dem Ternär-Operator bedingung ? wennJa : wennNein kannst du direkt im JSX entscheiden, was angezeigt wird. Für längere Logik nutze if/else vor dem return.

4. useState – Daten merken und ändern

Bisher haben unsere Komponenten nur Daten angezeigt, die von außen kamen (Props). Aber was, wenn sich etwas innerhalb der Komponente ändern soll? Dafür gibt es State (Zustand).

Stell dir den Unterschied so vor: Props sind wie ein Brief, den du bekommst – du kannst ihn lesen, aber nicht umschreiben. State ist wie eine Tafel in deinem Zimmer – du kannst jederzeit etwas Neues draufschreiben.

useState ist ein sogenannter Hook – eine spezielle Funktion von React. Er gibt dir zwei Dinge zurück: den aktuellen Wert und eine Funktion zum Ändern. 

function Counter() {
  const [count, setCount] = React.useState(0);
  //     ^^^^^  ^^^^^^^^                   ^
  //     Wert   Funktion zum Ändern       Startwert

  return (
    <div>
      <p>Zähler: {count}</p>
      <button onClick={function() { setCount(count + 1); }}>
        Plus 1
      </button>
    </div>
  );
}
count 0 → setCount(1) Klick! → count 1

setCount ändert den Wert. React rendert die Komponente dann automatisch neu.

Merke: useState gibt dir eine Variable und eine Funktion zum Ändern. Wenn du die Funktion aufrufst, wird die Komponente automatisch neu gerendert – die Anzeige aktualisiert sich von selbst!

5. Callback-Funktionen – Auf Aktionen reagieren

Du kennst schon addEventListener aus JavaScript. In React funktioniert das ähnlich, aber einfacher: Du schreibst Event-Handler direkt als Attribute in dein JSX.

Die wichtigsten Events: onClick (Klick), onChange (Änderung in Eingabefeldern) und onSubmit (Formular absenden). 

// onClick: Auf Klicks reagieren
function LikeButton() {
  const [likes, setLikes] = React.useState(0);

  function handleClick() {
    setLikes(likes + 1);
  }

  return (
    <button onClick={handleClick}>
      ❤️ {likes} Likes
    </button>
  );
}
// onChange: Eingaben verfolgen
function NameInput() {
  const [name, setName] = React.useState("");

  function handleChange(event) {
    setName(event.target.value);
  }

  return (
    <div>
      <input type="text" value={name} onChange={handleChange} />
      <p>Hallo, {name}!</p>
    </div>
  );
}

Ein besonders nützliches Muster: Du kannst eine Funktion als Callback-Prop an eine Kind-Komponente übergeben. So kann das Kind der Eltern-Komponente „Bescheid sagen“. 

function ColorButton(props) {
  return (
    <button onClick={function() { props.onSelect(props.color); }}>
      {props.color}
    </button>
  );
}

function ColorPicker() {
  const [selected, setSelected] = React.useState("Keine");

  return (
    <div>
      <p>Gewählte Farbe: {selected}</p>
      <ColorButton color="Rot" onSelect={setSelected} />
      <ColorButton color="Blau" onSelect={setSelected} />
      <ColorButton color="Grün" onSelect={setSelected} />
    </div>
  );
}

Callback-Kette:

ColorPicker (Eltern) – State: selected
ColorButton color="Rot" onSelect={setSelected}
Klick! → props.onSelect("Rot") → setSelected("Rot")
Merke: In JSX schreibst du onClick={funktionsname} – ohne Klammern! Mit Klammern würde die Funktion sofort aufgerufen. Callbacks als Props ermöglichen die Kommunikation vom Kind zur Eltern-Komponente.

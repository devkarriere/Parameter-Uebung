TypeScript, useEffect, useRef & Routing – Schritt für Schritt

Geführte Aufgabe: Die Tier-Steckbrief-App

Du baust eine App, die Tier-Steckbriefe anzeigt. Dabei lernst du TypeScript-Interfaces, useEffect zum Daten laden, useRef für den Fokus auf ein Suchfeld und einfaches Routing.

Vorbereitung: Erstelle ein neues Vite-Projekt mit TypeScript:
npm create vite@latest tier-steckbrief -- --template react-ts
cd tier-steckbrief
npm install
npm install react-router-dom
npm run dev
1

Das TypeScript-Interface für Tiere

Zuerst definierst du einen Bauplan für deine Tier-Daten. Erstelle eine Datei src/types/index.ts mit einem Interface, das beschreibt, welche Eigenschaften ein Tier hat: id (Zahl), name (Text), emoji (Text), habitat (Text), funFact (Text) und dangerous (Wahr/Falsch).

Lösung anzeigen 
// src/types/index.ts

export interface Animal {
  id: number;
  name: string;
  emoji: string;
  habitat: string;
  funFact: string;
  dangerous: boolean;
}
Beachte: Das export vor dem Interface macht es in anderen Dateien verfügbar. Ohne export könntest du es nicht importieren.

Die AnimalCard-Komponente mit typisierten Props

Erstelle eine Datei src/components/AnimalCard/AnimalCard.tsx. Die Komponente bekommt ein Tier-Objekt als Prop und zeigt Emoji, Name, Lebensraum und ob das Tier gefährlich ist. Wenn die Card geklickt wurde, soll ebenfalls eine übergeben Funktion mit der id als Parameter aufgerufen werden. Definiere ein eigenes Interface für die Props dieser Komponente.

Lösung anzeigen 
// src/components/AnimalCard/AnimalCard.tsx
import { Animal } from "../../types";

interface AnimalCardProps {
  animal: Animal;
  onClick: (id: number) => void;
}

function AnimalCard({ animal, onClick }: AnimalCardProps) {
  return (
    <div
      className="animal-card"
      onClick={function () { onClick(animal.id); }}
    >
      <span className="emoji">{animal.emoji}</span>
      <p className="name">{animal.name}</p>
      <p className="info">{animal.habitat}</p>
      {animal.dangerous ? (
        <p className="info" style={{ color: "#ef4444" }}>Gefährlich!</p>
      ) : null}
    </div>
  );
}

export default AnimalCard;
Animal Interface → animal Props → JSX Anzeige

Das Interface Animal wird im Props-Interface verwendet. TypeScript stellt sicher, dass alle Felder vorhanden sind.

Beachte: onClick ist hier eine Callback-Funktion als Prop. Der Typ (id: number) => void bedeutet: eine Funktion, die eine Zahl bekommt und nichts zurückgibt.

Home-Seite mit Tier-Karten

Erstelle die Datei src/views/Home/Home.tsx. Die Komponente soll:

  • Ein Array mit Tier-Daten als Konstante definieren
  • Für jedes Tier eine AnimalCard-Komponente anzeigen
  • Die Liste mit .map() durchgehen und die Karten rendern
Lösung anzeigen 
// src/views/Home/Home.tsx
import { Animal } from "../../types";

// Unsere Tier-Daten (später könnten die von einer API kommen)

Suchfeld mit Filter-Funktion

Füge ein Suchfeld zur Home-Komponente hinzu. Ein useState soll den Suchbegriff speichern und die Tier-Liste filtern.

Lösung anzeigen 
// Ergänze die Imports in Home.tsx:
import { useState } from "react";

// Innerhalb der Home-Funktion:
function Home() {
  const [searchTerm, setSearchTerm] = useState("");

  // Tiere filtern basierend auf Suchbegriff
  const filteredAnimals = animalData.filter(function (animal) {
    return animal.name.toLowerCase().includes(searchTerm.toLowerCase());
  });

  return (
    <div>
      <h2>Unsere Tiere</h2>
      <div className="search-bar">
        <input
          type="text"
          placeholder="Tier suchen..."
          value={searchTerm}
          onChange={function (e) { setSearchTerm(e.target.value); }}
        />
      </div>
      <div className="animal-grid">
        {filteredAnimals.map(function (animal) {
          return (
            <AnimalCard
              key={animal.id}
              animal={animal}
            />
          );
        })}
      </div>
    </div>
  );
}
Warum steht filteredAnimals einfach so in der Funktion – ohne useEffect?
Weil es eine Berechnung aus bestehendem State ist. Jedes Mal wenn sich searchTerm ändert, rendert React die Komponente neu – und dabei wird filteredAnimals automatisch neu berechnet. Du brauchst useEffect nur für Seiteneffekte – also Dinge, die außerhalb von React passieren (API-Aufrufe, Timer, Dokument-Titel ändern etc.). Eine Filterung ist kein Seiteneffekt, sondern einfach eine Ableitung aus dem State.
searchTerm ändert sich → Komponente rendert neu → filteredAnimals wird neu berechnet

State-Änderung → Re-Render → abgeleitete Werte werden automatisch aktualisiert. Kein useEffect nötig!

Detail-Seite mit Routing

Jetzt fügst du eine zweite Seite hinzu: Wenn du auf eine Tier-Karte klickst, öffnet sich die Detail-Ansicht mit dem Fun Fact. Dafür brauchst du:

  • Eine AnimalDetail-Komponente in src/views/AnimalDetail/AnimalDetail.tsx
  • Eine dynamische Route (/animal/:id)
  • useParams um die ID aus der URL zu lesen
  • Routing-Setup in App.tsx mit BrowserRouter
Lösung anzeigen 
// src/views/AnimalDetail/AnimalDetail.tsx
import { useParams, Link } from "react-router-dom";
import { Animal } from "../../types";

interface AnimalDetailProps {
  animals: Animal[];
}

function AnimalDetail({ animals }: AnimalDetailProps) {
  const { id } = useParams();

  const animal = animals.find(function (a) {
    return a.id === Number(id);
  });

  if (!animal) {
    return (
      <div>
        <p>Tier nicht gefunden!</p>
        <Link to="/">Zurück zur Übersicht</Link>
      </div>
    );
  }

  return (
    <div className="detail-view">
      <Link to="/">← Zurück</Link>
      <div style={{ textAlign: "center", marginTop: "1rem" }}>
        <span style={{ fontSize: "4rem" }}>{animal.emoji}</span>
        <h3 style={{ color: "#38bdf8", margin: "0.5rem 0" }}>{animal.name}</h3>
        <p>Lebensraum: {animal.habitat}</p>
        {animal.dangerous ? (
          <p style={{ color: "#ef4444" }}>⚠️ Dieses Tier ist gefährlich!</p>
        ) : (
          <p style={{ color: "#22c55e" }}>✅ Dieses Tier ist harmlos</p>
        )}
        <div className="fact" style={{
          background: "#1a2744",
          borderRadius: "8px",
          padding: "1rem",
          marginTop: "1rem"
        }}>
          <strong>Fun Fact:</strong> {animal.funFact}
        </div>
      </div>
    </div>
  );
}

export default AnimalDetail;
// src/App.tsx – Routing einrichten
import { createBrowserRouter, RouterProvider } from "react-router-dom";
import Home from "./views/Home/Home";
import AnimalDetail from "./views/AnimalDetail/AnimalDetail";

const router = createBrowserRouter([
  { path: "/", element: <Home /> },
  { path: "/animal/:id", element: <AnimalDetail /> },
]);

function App() {
  return <RouterProvider router={router} />;
}
Beachte: In der AnimalCard-Komponente musst du jetzt einen <Link> einbauen, damit bei Klick zur Detail-Seite navigiert wird: 
// In AnimalCard.tsx:
import { Link } from "react-router-dom";

function AnimalCard({ animal }: AnimalCardProps) {
  return (
    <Link to={"/animal/" + animal.id}>
      <div className="animal-card">
        <span>{animal.emoji}</span>
        <h3>{animal.name}</h3>
        <p>{animal.habitat}</p>
      </div>
    </Link>
  );
}

Layout mit Outlet

Zum Schluss erstellst du ein Layout mit Header und Footer, das auf jeder Seite sichtbar bleibt. Nutze eine Parent-Route mit <Outlet />.

Lösung anzeigen 
// src/components/Layout/Layout.tsx
import { Outlet, Link } from "react-router-dom";

function Layout() {
  return (
    <div>
      <header style={{
        textAlign: "center",
        padding: "1rem",
        borderBottom: "1px solid #334155"
      }}>
        <h1 style={{ color: "#38bdf8" }}>🐾 Tier-Steckbriefe</h1>
        <nav>
          <Link to="/" style={{ color: "#94a3b8", textDecoration: "none" }}>
            Startseite
          </Link>
        </nav>
      </header>

      <main style={{ maxWidth: "700px", margin: "2rem auto", padding: "0 1rem" }}>
        <Outlet />
      </main>

      <footer style={{ textAlign: "center", color: "#475569", marginTop: "3rem" }}>
        <p>Tier-Steckbrief-App © 2025</p>
      </footer>
    </div>
  );
}

export default Layout;
// Angepasste App.tsx mit Layout als Parent-Route
import { createBrowserRouter, RouterProvider } from "react-router-dom";
import Layout from "./components/Layout/Layout";
import Home from "./views/Home/Home";
import AnimalDetail from "./views/AnimalDetail/AnimalDetail";

const router = createBrowserRouter([
  {
    path: "/",
    element: <Layout />,
    children: [
      { index: true, element: <Home /> },
      { path: "animal/:id", element: <AnimalDetail /> },
    ],
  },
]);

function App() {
  return <RouterProvider router={router} />;
}

Komponenten-Hierarchie:

App – RouterProvider
Layout – Header, <Outlet />, Footer
Home (bei /) ODER AnimalDetail (bei /animal/:id)

Bonus: Sortierung & Zähler

Erweitere die App um zwei Funktionen:

  • Ein Dropdown zum Sortieren der Tiere (nach Name A-Z, Name Z-A)
  • Ein Zähler, der anzeigt, wie oft du einen Steckbrief angesehen hast (nutze useRef für einen Zähler, der kein Re-Render auslöst)
Lösung anzeigen 
// Tipp für den Zähler mit useRef (in AnimalCard):
const viewCountRef = useRef(0);

function handleClick() {
  viewCountRef.current = viewCountRef.current + 1;
  console.log("Insgesamt angesehen:", viewCountRef.current);
}

// Im JSX: <Link to={...} onClick={handleClick}>

// Tipp für die Sortierung (in Home):
const [sortOrder, setSortOrder] = useState<"az" | "za">("az");

const sortedAnimals = [...filteredAnimals].sort(function (a, b) {
  if (sortOrder === "az") {
    return a.name.localeCompare(b.name);
  }
  return b.name.localeCompare(a.name);
});
Beachte: useRef hat einen wichtigen Unterschied zu useState: Wenn du den .current-Wert änderst, wird die Komponente nicht neu gerendert. Das ist perfekt für Zähler oder andere Werte, die du nur im Hintergrund mitlaufen lassen willst.

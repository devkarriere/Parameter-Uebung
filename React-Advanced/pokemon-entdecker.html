Übung: Der Pokémon-Entdecker

Axios, Custom Hooks & dynamische Routen – alle Konzepte kombiniert

Komplexe Aufgabe: Der Pokémon-Entdecker

Du baust eine App, die echte Pokémon-Daten von der PokéAPI lädt und anzeigt. Dabei kombinierst du alle bisherigen Konzepte: Axios für API-Aufrufe, einen Custom Hook für die Datenlogik und React Router mit dynamischen Routen.

Vorbereitung: Erstelle ein neues Vite-Projekt:
npm create vite@latest pokemon-entdecker -- --template react-ts
cd pokemon-entdecker
npm install
npm install axios react-router-dom
npm run dev
API-Info: Die PokéAPI ist kostenlos und braucht keinen Schlüssel. Die wichtigsten Endpunkte:
https://pokeapi.co/api/v2/pokemon?limit=20 – Liste der ersten 20 Pokémon
https://pokeapi.co/api/v2/pokemon/1 – Details zu einem bestimmten Pokémon (nach ID)
1

TypeScript-Interfaces für die API-Daten

Erstelle die Typen in src/types/index.ts. Du brauchst:

  • PokemonListItem – für die Übersicht (Name + URL)
  • PokemonListResponse – für die API-Antwort der Liste (count + results-Array)
  • PokemonStat – für einen einzelnen Statuswert (base_stat + stat-Name)
  • PokemonType – für einen Typ-Eintrag (type-Name)
  • PokemonDetail – für die Detailseite (ID, Name, Bild, Typen, Stats)

Schau dir die API-Antworten an, um die Struktur zu verstehen. Du musst nicht alle Felder abbilden – nur die, die du anzeigen willst.

Lösung anzeigen 
// src/types/index.ts

// Für die Listen-Ansicht
export interface PokemonListItem {
  name: string;
  url: string;
}

export interface PokemonListResponse {
  count: number;
  results: PokemonListItem[];
}

// Für die Detail-Ansicht
export interface PokemonStat {
  base_stat: number;
  stat: {
    name: string;
  };
}

export interface PokemonType {
  type: {
    name: string;
  };
}

export interface PokemonDetail {
  id: number;
  name: string;
  sprites: {
    front_default: string;
    other: {
      "official-artwork": {
        front_default: string;
      };
    };
  };
  types: PokemonType[];
  stats: PokemonStat[];
  height: number;
  weight: number;
}
Tipp: Du musst nicht die gesamte API-Antwort abtippen. TypeScript beschwert sich nicht, wenn die API mehr Felder liefert als du definiert hast. Definiere nur die Felder, die du auch benutzt.

Axios einrichten

Erstelle eine Datei src/api/pokeApi.ts, in der du eine Axios-Instanz mit der Basis-URL der PokéAPI erstellst. So musst du die URL nicht bei jedem Aufruf wiederholen.

Lösung anzeigen 
// src/api/pokeApi.ts
import axios from "axios";

const pokeApi = axios.create({
  baseURL: "https://pokeapi.co/api/v2",
});

export default pokeApi;
Vorteil: Mit axios.create() erstellst du eine vorkonfigurierte Instanz. Statt axios.get("https://pokeapi.co/api/v2/pokemon") schreibst du nur noch pokeApi.get("/pokemon"). Das spart Tipparbeit und macht den Code übersichtlicher.

Custom Hook: usePokemonList

Erstelle den Hook src/hooks/usePokemonList.ts. Er soll:

  • Die ersten 20 Pokémon von der API laden (beim ersten Anzeigen)
  • Einen loading-State haben
  • Einen error-State haben
  • Die Liste der Pokémon zurückgeben
Lösung anzeigen 
// src/hooks/usePokemonList.ts
import { useState, useEffect } from "react";
import pokeApi from "../api/pokeApi";
import { PokemonListItem, PokemonListResponse } from "../types";

function usePokemonList() {
  const [pokemon, setPokemon] = useState<PokemonListItem[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState("");

  useEffect(function () {
    pokeApi.get<PokemonListResponse>("/pokemon?limit=20")
      .then(function (response) {
        setPokemon(response.data.results);
        setLoading(false);
      })
      .catch(function (err) {
        setError("Fehler beim Laden: " + err.message);
        setLoading(false);
      });
  }, []);

  return { pokemon, loading, error };
}

export default usePokemonList;
usePokemonList() Custom Hook → pokeApi.get() Axios-Anfrage → { pokemon, loading, error } Rückgabe

Der Custom Hook kapselt die gesamte Lade-Logik. Die Komponente ruft nur usePokemonList() auf und bekommt die Daten.

Custom Hook: usePokemonDetail

Erstelle einen zweiten Hook src/hooks/usePokemonDetail.ts. Er bekommt eine ID oder einen Namen als Parameter und lädt die Details des einzelnen Pokémon.

Lösung anzeigen 
// src/hooks/usePokemonDetail.ts
import { useState, useEffect } from "react";
import pokeApi from "../api/pokeApi";
import { PokemonDetail } from "../types";

function usePokemonDetail(nameOrId: string) {
  const [pokemon, setPokemon] = useState<PokemonDetail | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState("");

  useEffect(function () {
    setLoading(true);
    setError("");

    pokeApi.get<PokemonDetail>("/pokemon/" + nameOrId)
      .then(function (response) {
        setPokemon(response.data);
        setLoading(false);
      })
      .catch(function (err) {
        setError("Pokémon nicht gefunden: " + err.message);
        setLoading(false);
      });
  }, [nameOrId]);  // Neu laden wenn sich der Name/die ID ändert

  return { pokemon, loading, error };
}

export default usePokemonDetail;
Beachte: [nameOrId] im Abhängigkeits-Array sorgt dafür, dass die Daten neu geladen werden, wenn du ein anderes Pokémon aufrufst. Ohne dieses Array würde immer das erste Pokémon angezeigt bleiben.

Pokémon-Liste als View

Erstelle src/views/PokemonList/PokemonList.tsx. Die Seite nutzt den usePokemonList-Hook und zeigt eine Karte pro Pokémon. Jede Karte ist ein Link zur Detail-Seite. Für das Bild kannst du die URL-Formel nutzen: https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/{id}.png

Lösung anzeigen 
// src/views/PokemonList/PokemonList.tsx
import { Link } from "react-router-dom";
import usePokemonList from "../../hooks/usePokemonList";

// ID aus der URL extrahieren:
// "https://pokeapi.co/api/v2/pokemon/4/" → 4
function getIdFromUrl(url: string): number {
  const parts = url.split("/");
  return Number(parts[parts.length - 2]);
}

function PokemonList() {
  const { pokemon, loading, error } = usePokemonList();

  if (loading) return <p>Lade Pokémon...</p>;
  if (error) return <p style={{ color: "#ef4444" }}>{error}</p>;

  return (
    <div>
      <h2>Pokémon-Übersicht</h2>
      <div className="pokemon-grid">
        {pokemon.map(function (p) {
          const id = getIdFromUrl(p.url);
          return (
            <Link
              key={p.name}
              to={"/pokemon/" + id}
              style={{ textDecoration: "none" }}
            >
              <div className="pokemon-card">
                <img
                  src={"https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/" + id + ".png"}
                  alt={p.name}
                />
                <p className="id">#{id}</p>
                <p className="name">{p.name}</p>
              </div>
            </Link>
          );
        })}
      </div>
    </div>
  );
}

export default PokemonList;

Pokémon-Detail mit dynamischer Route

Erstelle src/views/PokemonDetail/PokemonDetail.tsx. Die Seite liest die ID aus der URL mit useParams, lädt die Details mit dem usePokemonDetail-Hook und zeigt: Bild, Name, Typen (als farbige Badges) und Statuswerte (HP, Angriff etc.) als Balken.

Lösung anzeigen 
// src/views/PokemonDetail/PokemonDetail.tsx
import { useParams, Link } from "react-router-dom";
import usePokemonDetail from "../../hooks/usePokemonDetail";

function getTypeClass(typeName: string): string {
  const typeMap: Record<string, string> = {
    fire: "type-fire",
    water: "type-water",
    grass: "type-grass",
    electric: "type-electric",
    normal: "type-normal",
    poison: "type-poison",
    bug: "type-bug",
    flying: "type-flying",
  };
  return typeMap[typeName] || "type-default";
}

function getStatColor(value: number): string {
  if (value >= 100) return "#22c55e";
  if (value >= 70) return "#f59e0b";
  if (value >= 40) return "#3b82f6";
  return "#ef4444";
}

function PokemonDetailView() {
  const { id } = useParams();
  const { pokemon, loading, error } = usePokemonDetail(id || "1");

  if (loading) return <p>Lade Details...</p>;
  if (error) return <p style={{ color: "#ef4444" }}>{error}</p>;
  if (!pokemon) return <p>Nicht gefunden</p>;

  return (
    <div>
      <Link to="/" style={{ color: "#94a3b8", textDecoration: "none" }}>
        ← Zurück zur Übersicht
      </Link>

      <div className="pokemon-detail" style={{ marginTop: "1rem" }}>
        <img
          src={pokemon.sprites.other["official-artwork"].front_default}
          alt={pokemon.name}
        />
        <h2>{pokemon.name} <span style={{ color: "#64748b" }}>#{pokemon.id}</span></h2>

        <div style={{ marginBottom: "1rem" }}>
          {pokemon.types.map(function (t) {
            return (
              <span key={t.type.name} className={"type-badge " + getTypeClass(t.type.name)}>
                {t.type.name}
              </span>
            );
          })}
        </div>

        <p style={{ color: "#94a3b8", marginBottom: "1rem" }}>
          Größe: {pokemon.height / 10}m | Gewicht: {pokemon.weight / 10}kg
        </p>

        <div style={{ maxWidth: 400, margin: "0 auto", textAlign: "left" }}>
          {pokemon.stats.map(function (s) {
            return (
              <div key={s.stat.name} className="stat-bar">
                <span className="stat-label">{s.stat.name}</span>
                <div className="stat-track">
                  <div
                    className="stat-fill"
                    style={{
                      width: Math.min(s.base_stat, 150) / 1.5 + "%",
                      background: getStatColor(s.base_stat)
                    }}
                  />
                </div>
                <span className="stat-value">{s.base_stat}</span>
              </div>
            );
          })}
        </div>
      </div>
    </div>
  );
}

export default PokemonDetailView;

Datenfluss bei der Detail-Seite:

URL: /pokemon/25 → useParams() → id = "25"
usePokemonDetail("25") → useEffect
pokeApi.get("/pokemon/25") → Pikachu-Daten → setPokemon()

Router Setup mit Layout

Erstelle zuerst eine Layout-Komponente in src/components/Layout/Layout.tsx mit Header, <Outlet /> und Footer. Baue dann in App.tsx den Router zusammen: Eine Index-Route für die Liste und eine dynamische Route für die Detail-Seite.

Lösung anzeigen 
// src/components/Layout/Layout.tsx
import { Outlet, Link } from "react-router-dom";

function Layout() {
  return (
    <div>
      <header style={{
        textAlign: "center",
        padding: "1rem",
        borderBottom: "1px solid #334155"
      }}>
        <Link to="/" style={{ textDecoration: "none" }}>
          <h1 style={{ color: "#f59e0b" }}>Pokémon-Entdecker</h1>
        </Link>
      </header>

      <main style={{ maxWidth: "700px", margin: "2rem auto", padding: "0 1rem" }}>
        <Outlet />
      </main>

      <footer style={{ textAlign: "center", color: "#475569", marginTop: "3rem" }}>
        <p>Pokémon-Entdecker © 2025 | Daten von PokéAPI</p>
      </footer>
    </div>
  );
}

export default Layout;
// src/App.tsx
import { createBrowserRouter, RouterProvider } from "react-router-dom";
import Layout from "./components/Layout/Layout";
import PokemonList from "./views/PokemonList/PokemonList";
import PokemonDetailView from "./views/PokemonDetail/PokemonDetail";

const router = createBrowserRouter([
  {
    path: "/",
    element: <Layout />,
    children: [
      { index: true, element: <PokemonList /> },
      { path: "pokemon/:id", element: <PokemonDetailView /> },
    ],
  },
]);

function App() {
  return <RouterProvider router={router} />;
}

export default App;

Bonus: Suchfunktion & Pagination

Erweitere die App um zwei Funktionen:

  • Suche: Ein Eingabefeld, das direkt ein Pokémon nach Namen sucht (z.B. „pikachu“). Nutze dafür die API: https://pokeapi.co/api/v2/pokemon/pikachu
  • Pagination: Buttons für „Nächste 20“ und „Vorherige 20“. Nutze den offset-Parameter der API: ?limit=20&offset=20
Achtung: Für die Pagination musst du deinen usePokemonList-Hook aus Schritt 3 ändern. Er bekommt jetzt einen offset-Parameter. Die getIdFromUrl-Hilfsfunktion aus Schritt 5 funktioniert weiterhin korrekt – sie liest die ID direkt aus der API-URL, egal auf welcher Seite du bist.
Lösung anzeigen
Erklärung: In Schritt 3 hat usePokemonList() immer die ersten 20 Pokémon geladen. Jetzt bekommt er einen offset-Parameter, der sagt: „Überspringe die ersten X Pokémon“. Dadurch wird [offset] zum Abhängigkeits-Array – bei jeder Änderung lädt der Hook neu. 
// src/hooks/usePokemonList.ts (ANGEPASST – ersetzt Schritt 3)
import { useState, useEffect } from "react";
import pokeApi from "../api/pokeApi";
import { PokemonListItem, PokemonListResponse } from "../types";

function usePokemonList(offset: number = 0) {
  const [pokemon, setPokemon] = useState<PokemonListItem[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState("");

  useEffect(function () {
    setLoading(true);
    pokeApi.get<PokemonListResponse>(
      "/pokemon?limit=20&offset=" + offset
    )
      .then(function (response) {
        setPokemon(response.data.results);
        setLoading(false);
      })
      .catch(function (err) {
        setError("Fehler: " + err.message);
        setLoading(false);
      });
  }, [offset]);  // Neu laden wenn offset sich ändert

  return { pokemon, loading, error };
}

export default usePokemonList;
Beachte: Wir nutzen weiterhin die getIdFromUrl-Hilfsfunktion aus Schritt 5. Dadurch funktioniert die ID-Berechnung auch nach dem Filtern korrekt – die ID kommt direkt aus der API-URL, nicht aus der Position im Array. 
// src/views/PokemonList/PokemonList.tsx (ANGEPASST)
import { useState } from "react";
import { Link } from "react-router-dom";
import usePokemonList from "../../hooks/usePokemonList";

// ID aus der URL extrahieren (gleiche Funktion wie in Schritt 5)
function getIdFromUrl(url: string): number {
  const parts = url.split("/");
  return Number(parts[parts.length - 2]);
}

function PokemonList() {
  const [offset, setOffset] = useState(0);
  const [searchTerm, setSearchTerm] = useState("");
  const { pokemon, loading, error } = usePokemonList(offset);

  if (loading) return <p>Lade Pokémon...</p>;
  if (error) return <p style={{ color: "#ef4444" }}>{error}</p>;

  const filtered = pokemon.filter(function (p) {
    return p.name.includes(searchTerm.toLowerCase());
  });

  return (
    <div>
      <h2>Pokémon-Übersicht</h2>

      <input
        className="search-input"
        type="text"
        placeholder="Pokémon suchen..."
        value={searchTerm}
        onChange={function (e) { setSearchTerm(e.target.value); }}
      />

      <div className="pokemon-grid">
        {filtered.map(function (p) {
          const id = getIdFromUrl(p.url);
          return (
            <Link
              key={p.name}
              to={"/pokemon/" + id}
              style={{ textDecoration: "none" }}
            >
              <div className="pokemon-card">
                <img
                  src={"https://raw.githubusercontent.com/PokeAPI/sprites/master/sprites/pokemon/" + id + ".png"}
                  alt={p.name}
                />
                <p className="id">#{id}</p>
                <p className="name">{p.name}</p>
              </div>
            </Link>
          );
        })}
      </div>

      <div className="pagination">
        <button
          onClick={function () { setOffset(offset - 20); }}
          disabled={offset === 0}
        >
          ← Vorherige
        </button>
        <span>Seite {offset / 20 + 1}</span>
        <button onClick={function () { setOffset(offset + 20); }}>
          Nächste →
        </button>
      </div>
    </div>
  );
}

export default PokemonList;
Hinweis zur Suche: Diese Lösung filtert die bereits geladene Liste auf dem Client. Das funktioniert für die aktuelle Seite (20 Pokémon). Für eine echte Suche über alle Pokémon könntest du alternativ die API direkt aufrufen: pokeApi.get("/pokemon/" + searchTerm) – das liefert ein einzelnes Pokémon nach exaktem Namen.

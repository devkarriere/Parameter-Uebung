Aufbauende Aufgabe: Die Einkaufsliste

Du baust eine Einkaufsliste mit Kategorien. Dabei verwendest du useReducer für die Listenlogik, useContext für ein Theme-System und MUI-Komponenten für das Styling.

Vorbereitung: Erstelle ein neues Vite-Projekt oder nutze dein bestehendes:
npm create vite@latest einkaufsliste -- --template react-ts
cd einkaufsliste
npm install
npm install @mui/material @emotion/react @emotion/styled
npm run dev
1

TypeScript-Typen für Items & Actions

Definiere in src/types/index.ts die Typen für ein Einkaufs-Item (mit id, name, category, done) und die möglichen Aktionen (Hinzufügen, Löschen, Umschalten). Die Kategorie soll ein fester Typ mit genau vier Möglichkeiten sein.

Lösung anzeigen 
// src/types/index.ts

export type Category = "Obst" | "Gemüse" | "Milchprodukte" | "Sonstiges";

export interface ShoppingItem {
  id: number;
  name: string;
  category: Category;
  done: boolean;
}

// Actions für den Reducer:
export type ShoppingAction =
  | { type: "ADD"; name: string; category: Category }
  | { type: "REMOVE"; id: number }
  | { type: "TOGGLE"; id: number };
Beachte: type Category = "Obst" | "Gemüse" | ... ist ein sogenannter Union Type. Er erlaubt nur genau diese vier Werte – schreibst du etwas anderes, zeigt TypeScript sofort einen Fehler. Das ist sicherer als string, weil Tippfehler sofort auffallen.

Der Reducer für die Einkaufsliste

Erstelle eine Datei src/reducer/shoppingReducer.ts. Der Reducer bekommt den aktuellen State (ein Array von Items) und eine Action. Je nach Action-Typ soll er:

  • ADD: Ein neues Item hinzufügen (mit automatischer ID)
  • REMOVE: Ein Item anhand der ID entfernen
  • TOGGLE: Den done-Status eines Items umschalten
Lösung anzeigen 
// src/reducer/shoppingReducer.ts
import { ShoppingItem, ShoppingAction } from "../types";

let nextId = 1;

function shoppingReducer(
  state: ShoppingItem[],
  action: ShoppingAction
): ShoppingItem[] {

  if (action.type === "ADD") {
    const newItem: ShoppingItem = {
      id: nextId,
      name: action.name,
      category: action.category,
      done: false,
    };
    nextId = nextId + 1;
    return [...state, newItem];
  }

  if (action.type === "REMOVE") {
    return state.filter(function (item) {
      return item.id !== action.id;
    });
  }

  if (action.type === "TOGGLE") {
    return state.map(function (item) {
      if (item.id === action.id) {
        return { ...item, done: !item.done };
      }
      return item;
    });
  }

  return state;
}

export default shoppingReducer;
dispatch { type: "ADD", ... } → Reducer berechnet neuen State → [...items] neues Array

Jede Action beschreibt was passieren soll. Der Reducer entscheidet wie.

Wichtig: Der Reducer verändert den alten State nie direkt! Stattdessen gibst du immer ein neues Array zurück. Deshalb nutzen wir [...state, newItem] statt state.push(newItem), und state.map() statt direkt item.done = true.

ThemeContext mit useContext

Erstelle einen ThemeContext in src/context/ThemeContext.tsx. Er soll:

  • Einen State für das Theme haben ("light" oder "dark")
  • Eine Funktion toggleTheme bereitstellen
  • Einen Provider und einen Custom Hook useTheme exportieren
Lösung anzeigen 
// src/context/ThemeContext.tsx
import { createContext, useContext, useState, ReactNode } from "react";

type Theme = "light" | "dark";

interface ThemeContextType {
  theme: Theme;
  toggleTheme: () => void;
}

const ThemeContext = createContext<ThemeContextType | null>(null);

function ThemeProvider({ children }: { children: ReactNode }) {
  const [theme, setTheme] = useState<Theme>("dark");

  function toggleTheme() {
    setTheme(function (current) {
      return current === "dark" ? "light" : "dark";
    });
  }

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

function useTheme() {
  const context = useContext(ThemeContext);
  if (context === null) {
    throw new Error("useTheme muss innerhalb von ThemeProvider sein");
  }
  return context;
}

export { ThemeProvider, useTheme };
Beachte: Der Custom Hook useTheme() ist ein Wrapper um useContext(ThemeContext). Das hat zwei Vorteile: (1) Du musst den Context nicht in jeder Datei importieren, und (2) die Fehlerprüfung stellt sicher, dass der Hook nur innerhalb des Providers verwendet wird.

AddItemForm mit MUI-Komponenten

Erstelle die Komponente src/components/AddItemForm/AddItemForm.tsx. Nutze MUI-Komponenten (TextField, Button, Select/MenuItem). Die Komponente bekommt eine onAdd-Callback-Funktion als Prop.

Lösung anzeigen 
// src/components/AddItemForm/AddItemForm.tsx
import { useState } from "react";
import { TextField, Button, Select, MenuItem } from "@mui/material";
import { Category } from "../../types";

interface AddItemFormProps {
  onAdd: (name: string, category: Category) => void;
}

function AddItemForm({ onAdd }: AddItemFormProps) {
  const [name, setName] = useState("");
  const [category, setCategory] = useState<Category>("Sonstiges");

  function handleSubmit() {
    if (name.trim() === "") return;  // Leere Einträge verhindern

    onAdd(name.trim(), category);
    setName("");  // Eingabefeld leeren
  }

  return (
    <div style={{ display: "flex", gap: "0.5rem", marginBottom: "1.5rem" }}>
      <TextField
        label="Neues Item"
        value={name}
        onChange={function (e) { setName(e.target.value); }}
        onKeyDown={function (e) {
          if (e.key === "Enter") handleSubmit();
        }}
        size="small"
        fullWidth
      />
      <Select
        value={category}
        onChange={function (e) { setCategory(e.target.value as Category); }}
        size="small"
        style={{ minWidth: 140 }}
      >
        <MenuItem value="Obst">🍎 Obst</MenuItem>
        <MenuItem value="Gemüse">🥦 Gemüse</MenuItem>
        <MenuItem value="Milchprodukte">🧀 Milch</MenuItem>
        <MenuItem value="Sonstiges">📦 Sonstiges</MenuItem>
      </Select>
      <Button variant="contained" onClick={handleSubmit}>
        Hinzufügen
      </Button>
    </div>
  );
}

export default AddItemForm;
Tipp: onKeyDown mit e.key === "Enter" erlaubt das Hinzufügen per Enter-Taste – eine kleine Verbesserung der Benutzerfreundlichkeit.

ShoppingItem-Komponente

Erstelle src/components/ShoppingItem/ShoppingItem.tsx. Jedes Item zeigt eine Checkbox (zum Abhaken), den Namen, die Kategorie als farbiges Badge und einen Löschen-Button. Nutze MUI Checkbox und IconButton oder einfache HTML-Elemente.

Lösung anzeigen 
// src/components/ShoppingItem/ShoppingItem.tsx
import { ShoppingItem as ShoppingItemType } from "../../types";

interface ShoppingItemProps {
  item: ShoppingItemType;
  onToggle: (id: number) => void;
  onRemove: (id: number) => void;
}

function ShoppingItemRow({ item, onToggle, onRemove }: ShoppingItemProps) {
  // Kategorie-Farben zuordnen
  function getCategoryClass(category: string): string {
    if (category === "Obst") return "cat-obst";
    if (category === "Gemüse") return "cat-gemuese";
    if (category === "Milchprodukte") return "cat-milch";
    return "cat-sonstiges";
  }

  return (
    <div className={"shopping-item" + (item.done ? " done" : "")}>
      <input
        type="checkbox"
        checked={item.done}
        onChange={function () { onToggle(item.id); }}
      />
      <span className="item-name">{item.name}</span>
      <span className={"item-category " + getCategoryClass(item.category)}>
        {item.category}
      </span>
      <button
        className="delete-btn"
        onClick={function () { onRemove(item.id); }}
      >
        ✕
      </button>
    </div>
  );
}

export default ShoppingItemRow;

Callback-Kette bei Abhaken:

App – useReducer(shoppingReducer, [])
ShoppingItemRow – onToggle={...}
Checkbox-Klick → onToggle(item.id) → dispatch({ type: "TOGGLE", id })

Alles zusammenbauen in App.tsx

Bringe alles zusammen: useReducer für die Items, den ThemeProvider um die App herum, und die Komponenten AddItemForm und ShoppingItemRow. Füge außerdem einen Zähler hinzu, der anzeigt, wie viele Items noch offen sind.

Lösung anzeigen 
// src/App.tsx
import { useReducer } from "react";
import { ThemeProvider, useTheme } from "./context/ThemeContext";
import shoppingReducer from "./reducer/shoppingReducer";
import AddItemForm from "./components/AddItemForm/AddItemForm";
import ShoppingItemRow from "./components/ShoppingItem/ShoppingItem";
import { Category } from "./types";

function ShoppingApp() {
  const [items, dispatch] = useReducer(shoppingReducer, []);
  const { theme, toggleTheme } = useTheme();

  function handleAdd(name: string, category: Category) {
    dispatch({ type: "ADD", name: name, category: category });
  }

  function handleToggle(id: number) {
    dispatch({ type: "TOGGLE", id: id });
  }

  function handleRemove(id: number) {
    dispatch({ type: "REMOVE", id: id });
  }

  const openCount = items.filter(function (i) { return !i.done; }).length;
  const doneCount = items.filter(function (i) { return i.done; }).length;

  return (
    <div style={{
      maxWidth: 500,
      margin: "2rem auto",
      padding: "0 1rem"
    }}>
      <h1>🛒 Einkaufsliste</h1>
      <button onClick={toggleTheme}>
        {theme === "dark" ? "☀️" : "🌙"}
      </button>

      <AddItemForm onAdd={handleAdd} />

      <div style={{
        display: "flex",
        gap: "1rem",
        marginBottom: "1rem",
        fontSize: "0.85rem",
        color: "#94a3b8"
      }}>
        <span>📋 Offen: {openCount}</span>
        <span>✅ Erledigt: {doneCount}</span>
      </div>

      {items.length === 0 ? (
        <p style={{ textAlign: "center", color: "#64748b" }}>
          Noch keine Items. Füge etwas hinzu!
        </p>
      ) : (
        items.map(function (item) {
          return (
            <ShoppingItemRow
              key={item.id}
              item={item}
              onToggle={handleToggle}
              onRemove={handleRemove}
            />
          );
        })
      )}
    </div>
  );
}

// App mit ThemeProvider umschließen
function App() {
  return (
    <ThemeProvider>
      <ShoppingApp />
    </ThemeProvider>
  );
}

export default App;
Beachte die Trennung: ShoppingApp ist die eigentliche App-Logik. Sie wird von App in den ThemeProvider eingewickelt. Das ist nötig, weil useTheme() nur innerhalb des Providers funktioniert.

Bonus: Filtern nach Kategorie

Füge einen Kategorie-Filter hinzu: Buttons für „Alle“, „Obst“, „Gemüse“, „Milchprodukte“ und „Sonstiges“. Nur Items der gewählten Kategorie werden angezeigt. Nutze dafür useMemo um die Filterung effizient zu machen.

Lösung anzeigen 
// Innerhalb von ShoppingApp ergänzen:
import { useReducer, useState, useMemo } from "react";

const [activeFilter, setActiveFilter] = useState<Category | "Alle">("Alle");

const filteredItems = useMemo(function () {
  if (activeFilter === "Alle") return items;
  return items.filter(function (item) {
    return item.category === activeFilter;
  });
}, [items, activeFilter]);

// Im JSX – Filter-Buttons:
<div style={{ display: "flex", gap: "0.5rem", marginBottom: "1rem", flexWrap: "wrap" }}>
  {["Alle", "Obst", "Gemüse", "Milchprodukte", "Sonstiges"].map(function (cat) {
    return (
      <button
        key={cat}
        onClick={function () { setActiveFilter(cat as Category | "Alle"); }}
        style={{
          background: activeFilter === cat ? "#2563eb" : "#334155",
          padding: "0.3rem 0.8rem",
          fontSize: "0.85rem"
        }}
      >
        {cat}
      </button>
    );
  })}
</div>

// Dann filteredItems statt items in der .map()-Schleife nutzen
